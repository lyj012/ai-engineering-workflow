# 独立评审 Rubric（质量门禁 / 最终评审）

> 由独立评审者执行（见 `.claude/agents/workflow-reviewer.md`）。实现者不得用本 rubric 自评。深度版见 `docs/08-evaluation-rubric.md`。

## 判级（硬性）
- **任何 P0 → FAIL**（无论分数）。
- 无 P0 但有阻断性 P1 / 关键证据缺失 → `CONDITIONAL_PASS`。
- 无 P0、无阻断 P1、证据齐全、分数达标 → `PASS`。

严重度：P0=需求未满足 / 不可运行 / 谎称已验证 / 有死循环 / 复用造成破坏；P1=影响质量或复用、有可行修复；P2=改进项。

## 100 分制权重
| 维度 | 分值 | 看什么 |
|---|---:|---|
| 需求覆盖与正确性 | 30 | 逐项对照原始需求；覆盖矩阵 met/partial/missing |
| 可运行性与真实证据 | 25 | 有真实运行记录与产物，不是“应该能跑” |
| 阶段/职责设计 | 15 | 每阶段输入/输出/完成标准；无重复无遗漏 |
| 健壮性 | 10 | 状态记录、失败重试、退出条件、无死循环、降级 |
| 复用现有 Skill | 10 | 复用而非复制，且未破坏现有资产 |
| 成本与规模克制 | 10 | 无为展示堆 agent、上下文/Token 可控 |

## 必须产出
1. verdict（PASS/CONDITIONAL_PASS/FAIL）
2. score（分项小计 + 总分）
3. 需求覆盖矩阵
4. P0/P1/P2 问题（每条含位置、建议、是否阻断）
5. 缺失证据清单
6. 必须返工项（P0/P1）
7. 重新通过条件
8. 一句话结论 + 是否建议向负责人展示

## 评审-返工流程（有界）
1. 全新实例评审 → 出 verdict。
2. 非 PASS：把 mustFix 发给**实现者**（不同实例）返工。
3. **新的**评审实例复评（重新独立核查，不信“已改”自述）。
4. 最多 2 轮返工；仍不过 → 如实输出未完成报告，不得声称完成。
