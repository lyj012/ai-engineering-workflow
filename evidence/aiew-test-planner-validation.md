# aiew_test_planner 单 Agent 验证记录

验证日期：2026-07-10
验证对象：`aiew_test_planner`
验证目标：验证单个 Agent 是否能为真实项目中的“任务创建功能”生成最小测试方案。

## 测试方案

| ID | 场景 | 类型 | 预期 | 优先级 |
|---|---|---|---|---|
| TC-001 | 提交完整且合法的标题、描述、任务类型、复杂度和风险标签 | 正常创建 | 请求成功；返回任务 ID；新增一条 `workflow_task` 记录 | P0 |
| TC-002 | 标题、任务类型或复杂度等必填字段为空或缺失 | 参数校验 | 请求失败并返回明确错误；不创建任务记录 | P0 |
| TC-003 | 创建任务后检查任务状态 | 状态初始化 | 任务记录进入系统定义的初始状态；状态与返回任务 ID 对应 | P0 |
| TC-004 | 提交带有一个或多个风险标签的任务 | 数据记录 | 返回成功；风险标签被正确保存，内容与请求一致 | P1 |
| TC-005 | 使用相同参数连续提交两次创建请求 | 重复提交风险 | 观察是否生成两条任务记录；若系统声明支持幂等，应返回同一任务或拒绝重复创建 | P1 |

## 本次不验证范围

- `template`、`matching`、`stage`、`prompt`、`delivery` 模块。
- 真实 AI 自动执行及执行结果。
- 完整 Workflow 阶段推进。
- 权限、认证、取消、重试、SSE 和并发压测。
- 复杂幂等策略、事务恢复和性能指标。

## 当前无法确认的项目事实

- 实际接口路径、HTTP 方法、请求字段名称及响应格式。
- 必填字段的精确定义和校验规则。
- 初始状态的具体枚举值。
- 风险标签的存储结构，是字符串、数组还是关联记录。
- 重复提交是否已有唯一约束或幂等机制。

原因：本次仅基于用户提供的项目事实和功能假设设计方案，未扫描项目、执行接口或检查数据库定义。以上内容需要在正式测试执行前补充确认。

## 验证结论

本次验证通过。

`aiew_test_planner` 可以作为单个 Agent 使用，能够输出质量合格的任务创建最小测试方案。它覆盖了正常创建、必填字段校验、状态初始化、风险标签保存和重复提交风险，并明确区分了已设计内容、不验证范围和无法确认的项目事实。

本次验证还证明：

- Agent 能正常响应。
- 输出测试用例数量控制在 5 个以内。
- 未修改文件。
- 未生成代码。
- 未执行真实接口或数据库测试。
- 未将假设描述为已确认事实。
- 未声称已经执行测试。

## 能力边界

本次仅证明 `aiew_test_planner` 能基于输入上下文设计合格的测试方案，尚未证明它能基于真实代码自动识别接口路径、字段定义、状态枚举和数据库结构。
